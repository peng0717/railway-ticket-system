## 1. 项目概述

### 1.1 项目名称

**模拟铁路车站人工售票系统 Web 版**（WebTRS - Web-based Railway Ticketing System）

本项目是 **sTRS（模拟售票系统 v5.0）** 的 Web 技术实现版本。

### 1.2 sTRS 参考说明

**sTRS（Simulated Ticket Reservation System）** 是一个模拟真实铁路客票发售和预订系统（TRS）的开源学习系统，完全参照中国铁路 TRS v5.0 的操作逻辑和界面设计。

sTRS 官方网站：https://strs.fun/

**sTRS 系统特征（WebTRS 完全继承）：**

| 特征类别 | 具体说明 |
|---------|---------|
| **三大业务模块** | 售票、退票、到达补票（工作站式操作界面，非自由导航的后台） |
| **登录流程** | 工号+密码登录 -> 选择班次（白班 00:00-12:00 / 夜班 12:00-24:00）-> 进入主界面 |
| **操作方式** | 键盘快捷键驱动为主（F1-F8功能键、Alt+字母、Ctrl+数字），鼠标为辅 |
| **站名输入** | 使用拼音码（如北京南=BJS，上海虹桥=SHH，郑州=ZZO），输入后弹出候选站名列表 |
| **售票流程** | F1日期 -> F2车次 -> F3发站 -> F4到站 -> F5票种张数 -> 选择席别 -> 回车确认 -> 输入收款 -> 印票 -> 输入旅客证件信息 |
| **票额用途** | F11切换（公用、机动、禁售、学生等） |
| **菜单结构** | 售票[G] 辅助[A] 制票[V] 处理[L] 查询[U] 交班[X] 帮助[B] |
| **出票方式** | 售票完成后生成车票图片（带"样票"水印），可下载保存 |
| **班次管理** | 到点自动提示结账交班，显示本班售票统计 |

### 1.3 项目目标

本项目旨在为编程学习者提供一套完整、可运行的铁路售票系统模拟平台。通过本项目，学员可以：

- 掌握 Flask Web 开发框架的实际应用
- 理解数据库设计与 CRUD 操作
- 熟悉业务流程设计与实现
- 体验从需求分析到系统交付的完整开发流程
- **学习参考 sTRS 系统的经典工作站式操作界面设计**

### 1.4 项目定位

| 定位维度 | 说明 |
|---------|------|
| **功能定位** | 模拟铁路车站人工售票窗口的核心业务流程，参照 sTRS 设计 |
| **用户定位** | 铁路售票员（模拟）、编程学习者 |
| **技术定位** | Flask + SQLite + Bootstrap 的 Web 应用，将 sTRS/TRS 桌面端转为 Web 版 |
| **操作定位** | 键盘驱动为主，完全参照 TRS v5.0 操作方式 |
| **部署定位** | 适合在 Replit 等免费云平台部署运行 |

### 1.6 与 sTRS 的关系

本项目 **WebTRS** 是 sTRS 的 Web 实现版本，参考 sTRS 的功能设计和操作方式，但使用不同的技术栈实现：

| 对比项 | sTRS（原版） | WebTRS（本项目） |
|-------|-------------|-----------------|
| 技术栈 | 桌面客户端 | Flask + SQLite + Bootstrap |
| 运行环境 | Windows | 浏览器（跨平台） |
| 部署方式 | 客户端安装 | Web 服务 |
| 数据存储 | 本地文件 | SQLite 数据库 |
| 操作方式 | 键盘+鼠标 | 键盘+鼠标（与 sTRS 一致） |
| 功能范围 | 专业版/标准版 | 基础售票功能（参照标准版） |


### 2.2 售票模块

#### 2.2.1 模块简介

售票模块是本系统的核心模块，完全参照 sTRS 的工作站式操作界面。售票员通过键盘快捷键驱动完成车次查询、旅客信息录入、票价计算、出票打印等操作。

#### 2.2.2 功能树状结构（与 sTRS 完全一致）

```
售票模块
|
+-- 售票子菜单 [G]
    |
    +-- 售普通票[S]              Alt+S
    +-- 售动车组车票[D]          Alt+D
    +-- 售通票[P]               Alt+P
    +-- --------------------------------
    +-- 通票中转签证
    +-- 中途下车签证
    +-- 始发签证[Z]             Alt+Z
    +-- 通票始发签证[C]          Alt+C
    +-- --------------------------------
    +-- 售优惠票[U]             Alt+U
    +-- 合同制票[Y]             Alt+Y
    +-- 进站补票[B]             Alt+B
    +-- 售往返联程票
    +-- --------------------------------
    +-- 挂失票处理
    +-- 用户核验/注销/预采集
    +-- 余票查询
```

| 输入项 | 说明 | 示例 |

| 票种 | 票种代码，全/孩/学 | 1/0/0 表示1张成人票 |
| 席别 | 席别类型 | 二等座、商务座、硬座等 |

**拼音码输入示例（必须实现）：**

| 站名 | 拼音码 | 说明 |
|------|--------|------|
| 北京 | BJ | 全拼首字母 |
| 北京南 | BJN | 全拼首字母 |
| 北京西 | BJX | 全拼首字母 |
| 上海虹桥 | SHH | 全拼首字母 |
| 上海南 | SHN | 全拼首字母 |
| 郑州 | ZZO | 郑州=ZhengZhou，Z+O特殊处理 |
| 石家庄 | SJZ | 全拼首字母 |
| 武汉 | WHN | 全拼首字母 |

**拼音码自动补全功能：**
- 输入时实时匹配候选站名列表
- 支持模糊匹配和精确匹配
- 上下键选择，回车确认

**TRS 输出项：**

| 输出项 | 说明 |
|--------|------|
| 车票信息 | 票号、车次、车厢、座位号、票价 |
| 仿真车票图片 | 内嵌火车票生成器，**带"样票"水印** |
| 下一张票号 | 显示下一张可用票号 |

**业务规则（与 sTRS/TRS 一致）：**
- 乘车站必须是车次经停站
- 到达站必须在乘车站之后
- 余票必须大于0
- **儿童票规则**：按年龄控制
  - 6周岁以下：免票（需申报免费儿童）
  - 6-14周岁：购买儿童票（成人票价的50%）
  - 14周岁以上：购买成人票
- **每名成人限带1名免费儿童**
- **购票时自动进行行程冲突校验**

**票价计算规则（简化版，与 sTRS 一致）：**

```
票价 = 基础票价 × 席别系数

席别系数：
- 硬座：1.0
- 软座：1.5
- 硬卧：1.8
- 软卧：2.5
- 二等座：1.8
- 一等座：2.8
- 商务座：4.5
```

**票号格式（7位，与真实TRS一致）：**
```
格式：1位字母 + 6位数字
示例：A043143、K001234

说明：
- 第1位：字母标识（与车站或类型相关）
- 第2-7位：流水号（6位数字）
```

##### （2）售通票

| 项目 | 内容 |
|------|------|
| **功能描述** | 发售需在中途站换乘的通票，可指定加快等级和径路 |
| **输入项** | 始发站、终到站、中转站、径路选择（最近径路/指定径路）、席别、加快等级（普通/加快/特快） |
| **输出项** | 通票信息（含经由站点的票价明细） |
| **业务规则** | - 中转站必须在始发站和终到站之间<br>- 径路必须经过中转站<br>- 通票票价 = 各段票价之和<br>- 加快费：普通=0、加快=票价×10%、特快=票价×20% |

##### （3）通票中转签证

| 项目 | 内容 |
|------|------|
| **功能描述** | 为已购买的通票办理中转换乘的签票手续 |
| **输入项** | 原通票信息、续乘日期、新车次、席别 |
| **输出项** | 签证后的新票信息 |
| **业务规则** | - 只能在原通票中转站办理<br>- 新车次必须经停中转站<br>- 席别不能低于原通票席别<br>- 签证费：免费 |

---

##### （4）中途下车签证

| 项目 | 内容 |
|------|------|
| **功能描述** | 旅客中途下车后，需要恢复旅行时办理的签证手续 |
| **输入项** | 原车票、新的乘车日期和新车次（可选） |
| **输出项** | 签证后的车票信息 |
| **业务规则** | - 只能在车票有效期内办理<br>- 如需换车次，补收或退还差价<br>- 签证费：2元/次 |

---

##### （5）始发签证（直达票改签）

| 项目 | 内容 |
|------|------|
| **功能描述** | 办理直达票的始发改签，可更改日期、车次、席别、到站等 |
| **输入项** | 原车票、新日期（可选）、新车次（可选）、新席别（可选）、新到站（可选） |
| **输出项** | 改签后的新车票 |
| **业务规则** | - 只能在开车前办理<br>- 同站改签免费，跨站改签需补差价<br>- 改签后票价高于原票价，补收差价<br>- 改签后票价低于原票价，退还差价<br>- 同级别席别可免费改签 |

---

##### （6）通票始发签证

| 项目 | 内容 |
|------|------|
| **功能描述** | 办理通票在始发站的改签，可更改日期、车次、席别、中转站、径路等 |
| **输入项** | 原通票、新日期、新车次、新席别、新中转站（可选）、新径路（可选） |
| **输出项** | 改签后的新通票 |
| **业务规则** | - 只能在始发站办理<br>- 更改径路需重新计算票价<br>- 其他规则同始发签证 |

##### （8）售优惠票

| 项目 | 内容 |
|------|------|
| **功能描述** | 发售动卧往返票、赠票、港九直通车、学生票、残军票等优惠票 |
| **输入项** | 票种类型、车次、乘车站、到站、席别、优惠原因 |
| **输出项** | 优惠票信息 |
| **业务规则** | - 动卧往返票：去程+返程同时购买，返程票价优惠20%<br>- 赠票：需单位证明，免收票价但收手续费5元<br>- 港九直通车：固定线路，票价按汇率计算<br>- **学生票：需先核验优惠资质，优惠区间内票价5折**<br>- **残军票：需先核验优惠资质，票价5折**<br>- **发售学生票、残军票时需先通过优惠资质核验** |

##### （9）售往返/联程票

| 项目 | 内容 |
|------|------|
| **功能描述** | 发售首程车票后，根据条件发售往返票或联程票 |
| **输入项** | 首程车票信息、返程/续程车次和日期 |
| **输出项** | 往返票/联程票信息 |
| **业务规则** | - 往返票：同一区间的往返程，返程优惠10%<br>- 联程票：不同区间的接续票，需要衔接时间合理<br>- 联程票必须连续使用 |

##### （10）进站补票/车票置换

| 项目 | 内容 |
|------|------|
| **功能描述** | 处理小孩无票补票、票种不符补票、车票换发等业务 |
| **输入项** | 旅客信息、原车票（如有）、补票原因 |
| **输出项** | 补票凭证或置换后的车票 |
| **业务规则** | - 无票乘车：补收票价+手续费50%<br>- 票种不符：补收差价+手续费10元<br>- 车票损坏：可免费换发 |

##### （11）挂失补票

| 项目 | 内容 |
|------|------|
| **功能描述** | 处理非电子客票（纸质票）丢失的补办业务 |
| **输入项** | 购票人证件信息、原车票信息（车次、日期、座位） |
| **输出项** | 挂失补办的新车票 |
| **业务规则** | - 只能在购票站办理<br>- 需提供有效证件<br>- 挂失手续费：2元<br>- 如原票已使用，新票票价按原票价计算 |

##### （12）合同制票

### 2.3 退票模块

#### 2.3.1 模块简介

退票模块负责处理各类退票业务，包括自愿退票、非自愿退票、电子客票退票等。

#### 2.3.2 功能树状结构

```
退票模块
├── 自愿退票
├── 非自愿退票（因故停运/延误）
├── 退电子客票
└── 退差价
```

#### 2.3.3 详细功能设计

---

##### （1）自愿退票

| 项目 | 内容 |
|------|------|
| **功能描述** | 旅客因个人原因主动办理退票 |
| **输入项** | 车票信息、旅客证件 |
| **输出项** | 退票凭证、退款金额 |
| **业务规则** | - 开车前15天以上：免收手续费<br>- 开车前48小时以上：收票价5%<br>- 开车前24-48小时：收票价10%<br>- 开车前24小时内：收票价20%<br>- 开车后：不予退票（改签除外） |

---

##### （2）非自愿退票

| 项目 | 内容 |
|------|------|
| **功能描述** | 因列车停运、严重晚点等原因为旅客办理退票 |
| **输入项** | 车票信息、停运/晚点证明 |
| **输出项** | 全额退款凭证 |
| **业务规则** | - 免收任何手续费<br>- 全额退还票价<br>- 可为后续改签提供优先权 |

---

##### （3）退电子客票

| 项目 | 内容 |
|------|------|
| **功能描述** | 退掉通过12306等平台购买的电子客票 |
| **输入项** | 订单号、旅客证件 |
| **输出项** | 退款凭证（原路返回） |
| **业务规则** | - 与普通退票规则相同<br>- 退款直接返回原支付渠道<br>- 需在开车前完成退票 |

---

##### （4）退差价

| 项目 | 内容 |
|------|------|
| **功能描述** | 因改签后票价降低，退还多收的差价 |
| **输入项** | 原车票、改签后新车票 |
| **输出项** | 差价退款凭证 |
| **业务规则** | - 仅退还票价差额<br>- 不收取任何手续费<br>- 需收回原车票（或确认已作废） |

### 3.2 数据表详细设计

---

#### 3.2.1 车站表（stations）

车站基础信息表，存储铁路车站的基本数据。

```sql
CREATE TABLE stations (
    station_id     INTEGER PRIMARY KEY AUTOINCREMENT,  -- 车站ID（自增主键）
    station_code   VARCHAR(6) NOT NULL UNIQUE,         -- 车站码（如 'BJN' 代表北京南）
    station_name   VARCHAR(50) NOT NULL,               -- 车站名称（如 '北京南'）
    station_pinyin VARCHAR(100),                       -- 车站拼音（如 'beijingnan'）
    region         VARCHAR(20),                        -- 所属省/市（如 '北京市'）
    line_name      VARCHAR(50),                        -- 所属线路（如 '京沪高铁'）
    is_major       BOOLEAN DEFAULT 0,                  -- 是否为主要站（1=是，0=否）
    status         VARCHAR(10) DEFAULT 'active',       -- 状态（active/inactive）
    created_at     TIMESTAMP DEFAULT CURRENT_TIMESTAMP -- 创建时间
);
```

**说明：**
- `station_code` 是车站的唯一标识代码，用于系统内部快速检索
- `pinyin_code` 是与 sTRS 一致的拼音码，用于拼音码输入（如 BJ=北京、BJN=北京南、ZZO=郑州）
- `is_major` 用于标记省会城市、直辖市等大站，便于查询显示优先级

**sTRS 拼音码规则：**
- 取车站名称全拼的首字母组合
- 特殊处理：如"郑州"=ZhengZhou，通常简写为 ZZO 或 ZZE
- 不唯一时加尾字母区分：如"南京"=NJ，"南京南"=NJN，"南京东"=NJD

---

#### 3.2.2 车次表（trains）

车次信息表，存储列车班次的基本数据。

```sql
CREATE TABLE trains (
    train_id       INTEGER PRIMARY KEY AUTOINCREMENT,  -- 车次ID（自增主键）
    train_number   VARCHAR(10) NOT NULL UNIQUE,        -- 车次号（如 'G1'、'K101'）
    train_type     VARCHAR(10) NOT NULL,               -- 车次类型（G=高铁/D=动车/K=普快/Z=直达）
    start_station  VARCHAR(50) NOT NULL,               -- 始发站
    end_station    VARCHAR(50) NOT NULL,               -- 终到站
    running_days   VARCHAR(7) DEFAULT '1234567',       -- 开行日（1-7代表周一到周日，1234567=每天都开）
    start_time     TIME,                               -- 始发时间
    end_time       TIME,                               -- 终到时间
    total_distance INTEGER,                            -- 总里程（公里）
    status         VARCHAR(10) DEFAULT 'active',       -- 状态（active/suspended/cancelled）
    created_at     TIMESTAMP DEFAULT CURRENT_TIMESTAMP -- 创建时间
);
```

**说明：**
- `train_type` 区分列车类型，不同类型票价系数不同
- `running_days` 支持设置每周哪几天开行，如"135"表示周一三五开行

---

#### 3.2.3 经停站表（train_stops）

车次经停站详情表，存储每个车次在各站的到发信息和余票数据。

```sql
CREATE TABLE train_stops (
    stop_id        INTEGER PRIMARY KEY AUTOINCREMENT,  -- 记录ID（自增主键）
    train_id       INTEGER NOT NULL,                   -- 车次ID（关联trains表）
    station_code   VARCHAR(6) NOT NULL,                -- 车站码（关联stations表）
    stop_sequence  INTEGER NOT NULL,                   -- 站序（1=始发站，依次递增）
    arrival_time  TIME,                               -- 到达时间（NULL表示始发站不适用）
    departure_time TIME,                               -- 出发时间（NULL表示终到站不适用）
    distance_from_start INTEGER,                      -- 距离始发站的里程（公里）
    
    -- 各席别余票数量
    seat_hard     INTEGER DEFAULT 0,                  -- 硬座余票
    seat_soft     INTEGER DEFAULT 0,                  -- 软座余票
    berth_hard    INTEGER DEFAULT 0,                   -- 硬卧余票
    berth_soft    INTEGER DEFAULT 0,                   -- 软卧余票
    seat_second   INTEGER DEFAULT 0,                  -- 二等座余票（动车/高铁）
    seat_first    INTEGER DEFAULT 0,                  -- 一等座余票（动车/高铁）
    seat_business INTEGER DEFAULT 0,                  -- 商务座余票（高铁）
    
    -- 各席别总票数（用于计算余票）
    total_hard     INTEGER DEFAULT 100,               -- 硬座总数
    total_soft     INTEGER DEFAULT 50,                -- 软座总数
    total_berth_hard INTEGER DEFAULT 60,              -- 硬卧总数
    total_berth_soft INTEGER DEFAULT 30,              -- 软卧总数
    total_second   INTEGER DEFAULT 500,               -- 二等座总数
    total_first    INTEGER DEFAULT 100,               -- 一等座总数
    total_business INTEGER DEFAULT 10,                -- 商务座总数
    
    FOREIGN KEY (train_id) REFERENCES trains(train_id),
    FOREIGN KEY (station_code) REFERENCES stations(station_code)
);
```

**说明：**
- 每条记录表示某车次在某个车站的停靠信息
- `arrival_time` 和 `departure_time` 为 NULL 时表示始发/终到站
- 余票字段支持按日期存储，这里简化设计，实际可扩展

---

#### 3.2.4 票价表（ticket_prices）

票价信息表，存储各区间的基准票价。

```sql
CREATE TABLE ticket_prices (
    price_id        INTEGER PRIMARY KEY AUTOINCREMENT, -- 票价ID（自增主键）
    from_station    VARCHAR(6) NOT NULL,              -- 出发站码
    to_station      VARCHAR(6) NOT NULL,              -- 到达站码
    seat_type       VARCHAR(20) NOT NULL,             -- 席别（hard/soft/berth_hard/berth_soft/second/first/business）
    base_price      DECIMAL(10,2) NOT NULL,          -- 基准票价（元）
    price_per_km    DECIMAL(10,4),                    -- 公里单价（元/公里，可选）
    effective_date  DATE DEFAULT (date('now')),       -- 生效日期
    FOREIGN KEY (from_station) REFERENCES stations(station_code),
    FOREIGN KEY (to_station) REFERENCES stations(station_code)
);
```

**说明：**
- 票价 = 基准票价，实际计算时可加入系数调整
- 可按日期维护不同时期的票价（节假日浮动等）

---

#### 3.2.5 车票表（tickets）

车票信息表，存储已售车票的完整信息。

```sql
CREATE TABLE tickets (
    ticket_id      VARCHAR(10) PRIMARY KEY,           -- 票号（7位格式：1位字母+6位数字，如 'A043143'、'K001234'）
    train_id       INTEGER NOT NULL,                  -- 车次ID
    train_number   VARCHAR(10) NOT NULL,              -- 车次号（冗余存储，便于查询）
    passenger_id   INTEGER,                           -- 旅客ID（关联passengers表）
    passenger_name VARCHAR(50),                       -- 旅客姓名（冗余存储）
    id_number      VARCHAR(30),                       -- 证件号码
    id_type        VARCHAR(10) DEFAULT 'id_card',     -- 证件类型（id_card/passport/hk_pass）
    
    -- 乘降信息
    from_station   VARCHAR(50) NOT NULL,              -- 出发站
    to_station     VARCHAR(50) NOT NULL,              -- 到达站
    travel_date    DATE NOT NULL,                     -- 乘车日期
    departure_time TIME NOT NULL,                     -- 出发时间
    
    -- 座位信息
    carriage       VARCHAR(5),                        -- 车厢号
    seat_number    VARCHAR(10),                       -- 座位号
    seat_type      VARCHAR(20) NOT NULL,              -- 席别
    
    -- 票价信息
    price          DECIMAL(10,2) NOT NULL,            -- 票价（元）
    ticket_type    VARCHAR(10) DEFAULT 'adult',     -- 票种（adult/child/student/disability）
    
    -- 分段接票信息
    segment_index  INTEGER DEFAULT 1,                -- 分段序号（1/2/3，用于分段接票）
    
    -- 免费儿童信息
    free_child_count INTEGER DEFAULT 0,              -- 随行免费儿童数量
    
    -- 状态和标识
    status         VARCHAR(20) DEFAULT 'valid',       -- 状态（valid/used/refunded/changed/lost）
    ticket_class   VARCHAR(20) DEFAULT 'normal',      -- 票类（normal=普通/transfer=通票/round=往返/concession=优惠）
    
    -- 售票信息
    seller_id      INTEGER,                           -- 售票员ID
    window_no      VARCHAR(10),                       -- 窗口号
    sold_at        TIMESTAMP DEFAULT CURRENT_TIMESTAMP, -- 售票时间
    
    -- 扩展信息
    endorsement_no VARCHAR(20),                       -- 改签票号（如果是由改签产生）
    remark         TEXT,                              -- 备注
    
    FOREIGN KEY (train_id) REFERENCES trains(train_id)
);
```

**说明：**
- `ticket_id` 采用7位格式（1位字母+6位数字），与真实TRS票号一致
- 第1位字母标识：与车站或票种相关（如 A=北京地区、K=普通票）
- 第2-7位：6位流水号
- `status` 字段用于追踪车票生命周期
- `endorsement_no` 用于关联改签前后的车票
- `segment_index` 用于分段接票场景，标识同一旅程中的分段序号
- `free_child_count` 记录该成人票随行的免费儿童数量（最多1名）

## 4. 界面设计

### 4.1 界面设计原则

```
+==========================================================================+
|                     界面设计原则（参照 sTRS/TRS v5.0）                      |
+==========================================================================+
|  1. 功能优先：以操作效率为核心，信息密度高，减少不必要的页面跳转         |
|  2. 键盘驱动：所有操作均可通过快捷键完成，保留鼠标操作支持              |
|  3. 专业严肃：参考真实TRS系统风格，配色沉稳，不做装饰性设计             |
|  4. 信息密集：核心操作界面一屏展示，售票员无需频繁滚动或切换页面         |
|  5. 工作站式：界面模拟售票窗口工作台，而非Web后台管理界面               |
|  6. 状态栏：底部固定显示快捷键提示和当前状态                            |
+==========================================================================+
```

**TRS v5.0 风格要素：**
- 深蓝紫渐变背景（登录页）
- 居中模态对话框（班次选择）
- 高信息密度表格布局
- 底部状态栏显示操作提示
- F1-F8 功能键行常驻显示
- 拼音码输入自动补全
- 键盘操作为主，鼠标为辅

## 5. 技术架构

### 5.1 技术栈选型

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| **后端框架** | Python Flask | 轻量级、学习曲线平缓，适合快速开发 |
| **数据库** | SQLite | 零配置、免安装、文件级数据库，适合Demo和小规模应用 |
| **前端框架** | Bootstrap 5 | 响应式设计、组件丰富、美观大方 |

### 5.3 拼音码输入组件设计

拼音码输入是 TRS 系统的核心功能，使用 JavaScript 实现自动补全：

```javascript
// 拼音码输入组件
class PinyinInput {
    constructor(inputElement, suggestionsContainer) {
        this.input = inputElement;
        this.container = suggestionsContainer;
        this.debounceTimer = null;

        // 监听输入事件
        this.input.addEventListener('input', (e) => {
            this.onInput(e.target.value);
        });

        // 监听键盘导航
        this.input.addEventListener('keydown', (e) => {
            if (e.key === 'ArrowDown') { this.selectNext(); e.preventDefault(); }
            if (e.key === 'ArrowUp') { this.selectPrev(); e.preventDefault(); }
            if (e.key === 'Enter') { this.confirmSelection(); e.preventDefault(); }
            if (e.key === 'Escape') { this.hideSuggestions(); }
        });
    }

    onInput(value) {
        // 防抖处理
        clearTimeout(this.debounceTimer);
        this.debounceTimer = setTimeout(() => {
            this.fetchSuggestions(value);
        }, 150);
    }

    async fetchSuggestions(pinyin) {
        if (pinyin.length < 1) { this.hideSuggestions(); return; }

        // 调用后端API获取候选站名
        const response = await fetch(`/api/stations/search?q=${pinyin}`);
        const stations = await response.json();

        this.showSuggestions(stations);
    }

    showSuggestions(stations) {
        // 渲染候选列表
        this.container.innerHTML = stations.map(s =>
            `<div class="suggestion-item" data-code="${s.pinyin_code}">
                ${s.station_name} (${s.pinyin_code})
            </div>`
        ).join('');

        this.container.style.display = 'block';
    }
}
```

**拼音码匹配规则：**
- 输入拼音码前缀，实时匹配
- 支持模糊匹配和精确匹配
- 返回结果按热度/优先级排序
- 选择后自动填充站名

### B. 火车票生成器集成方案

#### B.1 集成目标

在售票成功后，系统内直接展示仿真车票图片，售票员无需离开系统即可查看和下载车票图片，提升系统的真实感和体验感。

#### A.2 集成方式

**采用「内嵌集成 + 自动填充」方案**：将开源火车票生成器的前端构建产物嵌入本系统，售票成功后在系统内直接生成仿真车票，全程不跳转外部页面。

**源码仓库：** https://github.com/FoskyM/train-ticket-generator （AGPL-3.0 开源协议）

#### A.3 技术实现

**步骤一：构建前端产物**

从GitHub仓库获取源码，执行 `pnpm install && pnpm build`，生成 dist 静态文件目录。

**步骤二：嵌入Flask系统**

将 dist 目录内容放置到 Flask 项目的 `static/ticket-generator/` 目录下，通过 Flask 路由提供访问：

```python
@app.route('/ticket-preview/<ticket_id>')
def ticket_preview(ticket_id):
    ticket = Ticket.query.get(ticket_id)
    return render_template('ticket_preview.html', ticket=ticket)
```

```javascript
function autoFillTicket(ticketData) {
    const iframe = document.getElementById('ticket-generator');
    const doc = iframe.contentDocument;
    // 填充各字段
    doc.querySelector('[data-field="departure"]').value = ticketData.from_station;
    doc.querySelector('[data-field="destination"]').value = ticketData.to_station;
    doc.querySelector('[data-field="trainNumber"]').value = ticketData.train_number;
    // ... 其他字段
    // 触发渲染
    doc.querySelector('.generate-btn').click();
}
```
